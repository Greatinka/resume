# Разработка модели кредитного скоринга

## Проблематика

В ходе работы с кредитными данными выявлена проблема разнородности признаков: категориальные переменные (пол, тип жилья, цель кредита) и порядковые переменные (размер сберегательного счета) требовали принципиально различных подходов к предобработке.

## Оптимизация пайплайна обработки

Для обеспечения воспроизводимости и скорости обработки в production-среде реализовано следующее решение:

- Предварительная подготовка и сериализация энкодеров (OneHotEncoder, OrdinalEncoder) с помощью joblib.dump
- Сохранение StandardScaler для унифицированной нормализации поступающих данных
- Загрузка готовых трансформеров в production-среде, обеспечивающая обработку новых заявок за миллисекунды

## Выбор модели машинного обучения

В процессе экспериментов протестированы следующие классификаторы:

- RandomForestClassifier
- GradientBoostingClassifier (с оптимизацией гиперпараметров через GridSearchCV)

Наилучшие и наиболее стабильные результаты на тестовой выборке продемонстрировал градиентный бустинг.

## Особенности предобработки данных

Критически важным этапом стала корректная обработка категориальных признаков:

- Для порядковой переменной "Saving accounts" применен OrdinalEncoder с иерархией градаций: unknown → little → moderate → quite rich → rich
- Для номинативных признаков использован OneHotEncoder, исключающий внесение ложных порядковых отношений

## Оптимизация гиперпараметров

Выявлен немонотонный характер зависимости качества модели от гиперпараметров: увеличение количества деревьев с 400 до 500 не гарантировало прироста метрик. Применение GridSearchCV с кросс-валидацией позволило определить оптимальную комбинацию параметров n_estimators, max_depth и min_samples_leaf.

## Разведочный анализ данных

Перед обучением модели проведен комплексный анализ данных:

- Построение распределений возрастов и сумм кредита для понимания структуры данных
- Анализ корреляционной матрицы для выявления мультиколлинеарности
- Верификация независимости признаков, сгенерированных OneHotEncoder
