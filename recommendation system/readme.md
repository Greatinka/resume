# Разработка рекомендательной системы с высокой нагрузкой

## Цель проекта

Создание рекомендательной системы, способной выдерживать высокую интенсивность запросов и обеспечивающей качество не ниже 0.7 по метрике HitRate5 (наличие хотя бы одного лайка в пяти рекомендациях).

## Оптимизация производительности

Для обеспечения устойчивости к высокой нагрузке были реализованы следующие решения:

- Предварительная загрузка актуальных данных с их последующим использованием в оперативной памяти
- Минимизация операций слияния (merge) данных, требующих значительных вычислительных ресурсов
- Оптимизация обрабатываемого объема данных

## Выбор модели машинного обучения

В процессе разработки были протестированы различные подходы:

- XGBoost
- LGBMRanker
- Методы матричной классификации

Наилучшие результаты показало использование CatBoost, который также обеспечил удобство работы и позволил повысить качество модели за счет удаления незначимых признаков.

## Ключевое решение

Критическим фактором достижения целевого качества стала ребалансировка классов при обучении. В ходе экспериментов с десятью различными вариантами соотношения классов оптимальным оказалось соотношение 25% положительных примеров (лайков) к 75% отрицательных (просмотров).

Важно отметить немонотонный характер зависимости качества от дисбаланса классов, что допускает существование более оптимального решения при иных фиксированных параметрах.

## Технологический стек

- **Язык программирования:** Python

- **Обработка данных:** Pandas
- **Машинное обучение:** CatBoost
- **API-фреймворк:** FastAPI
- **Валидация данных:** Pydantic
