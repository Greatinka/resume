История создания сервиса:
Стояла задача создания рекомендательной системы, способной выдержать бомбардировку запросами и 
обеспечивающей качество 0.7 по метрике HitRate5 (хотя бы один лайк на пять предложений = 1, иначе 0).

Как пережить бомбардировку запросами:
Заранее загружаем наиболее свежие данные, и работаем с сохраненной информацией (много загружать нельзя - 
долго). Отказываемся от затратных по времени и памяти merg-ей и стараемся обрабатывать минимальное 
количество данных.

Как достичь нужного качества:
При реализации проекта перепробовал многие классификаторы: RandomForestClassifier, XGBoost, LGBMRanker,
а также матричную классификацию. Однако наилучший результат в моих опытах был достигнут при помощи 
Catboost, так же данный инструмент было легко и удобно использовать (При его помощи я удалил "неважные" фичи и 
повысил качество). Но ключевым моментом в решении поставленной задачи стала ребалансировка классов при 
обучении. Перепробовал 10 вариантов и по итогу мой выбор остановился на 25-75 (25% лайков и 75% просмотров)
Именно при данном соотношении в моих опытах достигалось наибольшее качество (Важно упомянуть, что при 
увеличении разницы между просмотрами и лайками качество росло немонотонно, поэтому возможно при 
фиксированных остальных параметрах я нашел не самое оптимальное решение).

Итоговый стек для выполнения проекта:
Python + PostgreSQL + Pandas + CatBoost + FastAPI + Pydantic
